Метод 11 — подробно
Анализ влияния человеческого фактора на надёжность системы

Метод оценивает влияние действий работника на работу рассматриваемой системы — в том числе ошибок, вызванных перегрузками. Позволяет идентифицировать ошибки и их причины, чтобы снизить количество ошибок и тяжесть последствий.

Адаптация: исходный метод (ГОСТ Р МЭК 62508‑2014, ГОСТ Р 58771‑2019) сфокусирован на операторских ошибках при управлении оборудованием. Мы переносим тот же инструментарий на психосоциальные и психофизические перегрузки — стресс, выгорание, когнитивную усталость, конфликтность.

Область применения: конкретное рабочее место или процесс, среднесрочный и долгосрочный диапазон, операционный и тактический уровень решений.

Нормативная база
ГОСТ Р МЭК 62508‑2014 · ГОСТ Р 58771‑2019 (Б.5.8) · ГОСТ Р 51901.1‑2002 · Приказ Минтруда № 926, п. 73–75 · Приложение № 5 «Стрессы на работе»
Приказ Минтруда № 926 п. 73–75, прил. 5
Шаг 1
Классификация факторов перегрузок

Психосоциальныеобусловлены организацией труда и социальным окружением:

  • жёсткие дедлайны, быстрый темп, большой объём работы
  • ненормированный рабочий день, отсутствие автономии
  • социальная изоляция, конфликты, агрессия, буллинг
  • недостаток поддержки руководства и обратной связи

Психофизическиесвязаны с нагрузкой на организм:

  • монотонность, гиподинамия, сенсорная депривация
  • информационная перегрузка, длительная концентрация внимания
  • сменная работа, нарушение циркадных ритмов
  • физическое истощение при тяжёлом труде
Результат шага
Реестр факторов перегрузок по каждому рабочему месту с отметкой: психосоциальный / психофизический / комбинированный
Приказ Минтруда № 926 прил. 5, пп. 73–75
Шаг 2
Типы ошибок человека при перегрузках

Ошибки по невниманию — пропуск сигнала, перепутанный тумблер. Возникают при монотонности, сенсорной депривации, усталости. Работник знает правильное действие, но не выполняет его.

Ошибки по памяти — забыл шаг алгоритма, не вспомнил инструкцию. Провоцируются информационной перегрузкой и стрессом, которые снижают рабочую память.

Ошибки по решению — выбран неверный план действий при нештатной ситуации. Вызваны дефицитом времени, когнитивной перегрузкой, отсутствием автономии.

Нарушения (умышленные) — отступление от правил ради экономии времени. Часто следствие хронического стресса, выгорания, конфликтов с руководством.

Результат шага
Карта ошибок: для каждого рабочего места — доминирующий тип ошибки и связанный с ней фактор перегрузки
Приказ Минтруда № 926 п. 73
Шаг 3
Оценка вероятности ошибки

Для каждого выявленного типа ошибки оценивается вероятность её совершения с учётом перегрузок. Используется модифицированная шкала из ГОСТ Р МЭК 62508:

  • Очень низкая (1) — ошибка практически исключена, перегрузки отсутствуют
  • Низкая (2) — ошибка маловероятна, эпизодический стресс
  • Средняя (3) — ошибка возможна, периодические перегрузки
  • Высокая (5) — ошибка вероятна, хронический стресс или выгорание
  • Очень высокая (7) — ошибка почти неизбежна, критическая перегрузка

Корректирующие множители: время суток (ночная смена ×1.3), стаж (<1 года ×1.2), отсутствие поддержки ×1.4, конфликтная среда ×1.3.

Результат шага
Базовая вероятность ошибки + корректирующий множитель = итоговая вероятность для каждого типа ошибки на рабочем месте
Приказ Минтруда № 926 прил. 13, ГОСТ Р МЭК 62508
Шаг 4
Оценка тяжести последствий

Последствия ошибки классифицируются по шкале тяжести из Приложения № 13 к Рекомендациям, с адаптацией под психологический ущерб:

  • Незначительный (1) — микротравма, дискомфорт, кратковременный стресс
  • Малый (5) — незначительное воздействие на здоровье, восстановление за 1–3 дня
  • Средний (10) — неблагоприятное воздействие, временная нетрудоспособность до 30 дней, начальные признаки профессионального заболевания
  • Большой (13) — значительная утрата трудоспособности, длительное расстройство 30–60 дней
  • Очень большой (15) — смертельный случай, хроническое заболевание, стойкая утрата трудоспособности
Результат шага
Значение тяжести последствий для каждого типа ошибки, включая психологический и психосоматический ущерб
Приказ Минтруда № 926 прил. 13
Шаг 5
Расчёт уровня риска

Уровень риска = Вероятность × Тяжесть. Результат сравнивается с порогами из Приложения № 13:

  • Низкий — риск удовлетворительный, дополнительные меры не требуются, поддерживайте текущий уровень
  • Умеренный — запланируйте мероприятия по снижению, установите сроки выполнения
  • Высокий — риск недопустим: немедленное устранение или приостановка работ до снижения

При комбинированных перегрузках (психосоциальная + психофизическая) вероятность ошибки суммарно выше, чем по каждому фактору отдельно. Рекомендуется применять поправочный коэффициент ×1.2 для учёта взаимного усиления.

Результат шага
Числовое значение риска по каждой ошибке; категория (низкий / умеренный / высокий); поправка за комбинированные перегрузки
Приказ Минтруда № 926 прил. 13, табл. 13.3
Шаг 6
Мероприятия по снижению риска

Организационные

  • исключить «пиковые» перегрузки, сбалансировать рабочую нагрузку
  • заблаговременно информировать о напряжённых периодах
  • предоставить автономию: гибкий график, участие в планировании
  • внедрить наставничество и регулярную обратную связь

Социальные

  • развивать способы разрешения конфликтов, политику противодействия агрессии
  • избегать работы в одиночку, содействовать командному взаимодействию
  • обучать руководителей навыкам поддержки и коммуникации

Медицинскиерегулярные проверки психологического состояния в периоды повышенной нагрузки.

Результат шага
План мероприятий по управлению рисками (прил. 16) с указанием меры, источника финансирования, срока и ответственного
Приказ Минтруда № 926 п. 87–93, прил. 5, 16
Итог
Документирование и интеграция

Результаты оценки фиксируются в перечне (реестре) рисков, ранжированном по уровню, и в плане управления рисками (прил. 16). Для каждой ошибки указаны:

  • фактор перегрузки (психосоциальный / психофизический)
  • тип ошибки (невнимание / память / решение / нарушение)
  • вероятность, тяжесть, уровень риска
  • корректирующий множитель и поправка на комбинированные перегрузки
  • запланированное мероприятие, срок, ответственный

Интеграция: количественные данные об ошибках оператора используются как исходные данные для FMEA, дерева отказов и дерева событий. После реализации мероприятий — повторная оценка риска (п. 91–93).

Итоговый документ
Реестр рисков + план мероприятий + протокол повторной оценки. Обоснование для СОУТ, обеспечения СИЗ и управленческих решений по ограничению перегрузок
Приказ Минтруда № 926 п. 74, 87–93, прил. 16
